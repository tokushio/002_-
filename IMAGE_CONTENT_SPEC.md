## 1. 画像永続化（バックエンド）

### 目的
InstagramのCDN画像はホットリンク拒否（403）でブラウザから表示できない。
サーバー側で取得してSupabase Storageに保存することで永続URLを発行し、
全デバイスから安定表示できるようにする。

### 処理フロー（Node.js 記事生成エンジン）

```
Instagram投稿を取得
        ↓
カルーセル画像を最大5枚サーバー側でfetch
（Node.jsからは取得可能なことを確認済み）
        ↓
Supabase Storageの永続バケットにアップロード
        ↓
発行された永続パブリックURLをDBに保存
articles.thumbnail_url    ← 1枚目を記事サムネイルに
article_parts[n].image_url ← n枚目を各パーツに紐づけ
```

### 画像のパーツへの分配ルール

| パーツ番号 | 使用する画像 |
|-----------|-------------|
| パーツ① | カルーセル1枚目 |
| パーツ② | カルーセル2枚目 |
| パーツ③ | カルーセル3枚目 |
| パーツ④以降 | カルーセル4・5枚目 |
| 元投稿が1枚のみ | パーツ①のみ画像表示。②以降はテキストのみ（ミニマルスタイル） |

### Supabase Storage設定
- バケット名：`article-images`
- アクセス：Public（パブリック読み取り）
- ファイル名：`{article_id}/{part_number}.jpg`

## 2. コンテンツ生成ルール（AIエンジン）

### 絶対禁止：元の投稿・施主への批判

以下のようなスタンスは**システムレベルで完全禁止**。

| ❌ NGスタンス | ⭕️ OKスタンス |
|-------------|--------------|
| 「この間取りは失敗しています」 | 「一般論として通路幅が狭いと渋滞リスクがあります」 |
| 「この施工は後悔します」 | 「プロの推奨基準は〇〇cm以上です」 |
| 「ここがダメ」 | 「より確実に成功させるための補足ポイント」 |

### 統一トーン：上品な教育コラム

元の投稿を「叩く」のではなく、
**「素晴らしいアイデアをさらに成功させるためのプロの補足解説」**
という文脈に100%統一する。

## 3. クレジット表示・ディープリンク（フロントエンド）

### 目的
一次情報へのリスペクトを表明し、
元の投稿者に**フォロワー増・インプレッション増**という実利を返すことで
著作権クレーム・通報リスクをゼロに近づける。

### 各パーツカード最下部に必ず表示するUI

```
┌──────────────────────────────┐
│ 【パーツタイトル】             │
│ ① 核心的な価値               │
│ ② 数値・条件・体感ポイント    │
│ ③ 次の行動                   │
│ [画像]                       │
│ ─────────────────────────── │
│ 📌 このアイデアの元投稿を見る  │ ← 必須
│    （Instagramアプリが開きます）│
└──────────────────────────────┘
```

### ディープリンクの技術仕様

```
// iOSのInstagramアプリを直接起動
instagram://media?id={media_id}

// Instagramアプリ未インストールの場合のフォールバック
https://www.instagram.com/p/{shortcode}/

// 実装（React Native）
const handleOpenInstagram = async () => {
  const deepLink = `instagram://media?id=${mediaId}`
  const webLink = `https://www.instagram.com/p/${shortcode}/`
  const canOpen = await Linking.canOpenURL(deepLink)
  Linking.openURL(canOpen ? deepLink : webLink)
}
```

## 4. オプトアウト（掲載取り下げ）機能

### 目的
施主・インフルエンサーから取り下げ要望が来た場合に
SNS炎上・ストア通報に発展する前にアプリ内で即時解決する。

### アクセス導線
- アプリ内メニューから1タップでアクセス
- 各パーツカードのインフォメーション（ℹ️）ボタンからも到達可能

### フォームの文面
```
もし掲載の取り下げ、または画像の非表示をご希望の場合は、
こちらからご連絡ください。迅速に対応いたします。
```

### バックエンドのステータス管理

```sql
-- materialsテーブルに追加するカラム
is_hidden BOOLEAN DEFAULT FALSE,
hidden_at TIMESTAMPTZ,
hidden_reason TEXT
```

```
管理者が is_hidden = true に設定
        ↓
フロントエンドが即座に
・画像 → ダミー画像（グレープレースホルダー）に差し替え
・「元投稿を見る」リンク → 非表示
        ↓
申請者への返信：「対応しました」
```

### 対応SLA
- 受信から**24時間以内**に対応・非表示完了
